# Session Log Skill

每次用户与 Doko 的交互即将结束时，你必须生成一份会话日志文件。

## 触发时机

当你判断用户的任务已经完成（用户表示满意、说"谢谢"、没有后续问题、或者你已完成所有修改），在给出最终回复之前，执行以下步骤。

## 日志文件规范

### 文件位置与命名
- 目录：`.doko/logs/`（工作目录下）
- 文件名格式：`YYYY-MM-DD_HH-mm-ss.md`（使用当前时间戳）
- 如果目录不存在，先用 `execute_bash` 创建

### 文件内容模板

```markdown
# Doko Session Log

- **日期**: YYYY-MM-DD HH:mm:ss
- **耗时**: 大约 N 分钟（估算）
- **主题**: 一句话概括本次做了什么

## 任务摘要

用 2-5 句话描述用户的需求和最终达成的结果。

## 操作记录

按时间顺序列出关键操作：

1. [读取] 读取了 `path/to/file` 了解现有结构
2. [创建] 创建了 `path/to/new-file` 实现 XXX 功能
3. [修改] 修改了 `path/to/file` 添加了 XXX
4. [执行] 运行了 `command` 验证 XXX

## 涉及文件

| 文件路径 | 操作 | 说明 |
|---------|------|------|
| `src/Foo.java` | 新增 | 实现了 XXX |
| `src/Bar.java` | 修改 | 添加了 YYY |

## 备注

记录任何需要后续关注的事项，例如：
- 未完成的 TODO
- 已知的限制或风险
- 建议的后续步骤
```

## 注意事项

- 只记录事实，不要编造没有发生的操作
- 文件路径使用相对路径
- 操作类型标签使用：[读取] [创建] [修改] [删除] [执行]
- 如果本次会话没有做任何实质修改（只是问答），也要记录，主题写"问答/咨询"
- 使用 `execute_bash` 获取当前时间来生成文件名：`date '+%Y-%m-%d_%H-%M-%S'`
